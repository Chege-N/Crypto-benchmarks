
DSA File Signing and Verification Benchmark (2048-Bit Key)

This Python script demonstrates the generation of a 2048-bit DSA (Digital Signature Algorithm) key pair using the pycryptodome library, followed by hashing, signing, and verifying the signatures for small (1KB) and large (1MB) files. 
It measures and prints the execution time for key generation and the signing/verification processes.The script uses SHA-256 hashing and DSA in FIPS-186-3 mode for secure, deterministic signing. It verifies the signatures immediately and outputs "OK" if valid.

Requirements
Python 3.x,
pycryptodome library (install via pip install pycryptodome)

Installation
1.Clone or download the script to your local machine.
2.Install the required library:
pip install pycryptodome

Usage
1.Prepare input files:
Create a small text file named smallFile_1KB.txt (approximately 1KB in size).
Create a large text file named largeFile_1MB.txt (approximately 1MB in size).
2.Run the script:
dsa-2048.py

4.The script will:
Generate a 2048-bit DSA key pair and print the generation time.
Hash, sign, and verify the small file, printing "OK" if valid and the process time.
Hash, sign, and verify the large file, printing "OK" if valid and the process time.
Note: Times are printed in actual microseconds (corrected from previous versions). Example output:
2048 bit DSA key generation time in microseconds: 5637798.6000152305OK
Using 2048 bit DSA key signing and verification time in microseconds: 6775.40001925081
OK
Using 2048 bit DSA key signing and verification time in microseconds: 12694.700038991868

Script Overview
Key Generation: Uses DSA.generate(2048) to create the key pair.
Hashing: Reads files in 4KB chunks and updates a SHA-256 hash using Crypto.Hash.SHA256.
Signing: Creates a signer with DSS.new(key, 'fips-186-3') and signs the hash object.
Verification: Creates a verifier with DSS.new(pub_key, 'fips-186-3') and verifies the signature against the hash.
Limitations
DSA is used for digital signatures only (integrity and authenticity), not encryption.
Signing/verification time is near-constant (operates on the hash), but hashing large files adds overhead.
No error handling for file I/O issues beyond basic verification.
Assumes input files exist; the script will fail otherwise.
DSA is considered legacy; for modern applications, prefer ECDSA or EdDSA for better performance and security.
