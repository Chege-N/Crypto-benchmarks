
RSA File Encryption Benchmark

This Python script demonstrates the generation of a 2048-bit RSA key pair using the pycryptodome library, followed by the encryption and decryption of small (1KB) and large (1MB) files.
It measures and prints the execution time for key generation, encryption, and decryption operations.The script processes files in chunks to accommodate RSA's block size limitations and uses PKCS1_OAEP padding for secure encryption/decryption.

Requirements
1.Python 3.x,
2.pycryptodome library (install via pip install pycryptodome)

Installation
1.Clone or download the script to your local machine.
2.Install the required library:
pip install pycryptodome

Usage
1.Prepare input files:
Create a small text file named smallFile_1KB.txt (approximately 1KB in size).
Create a large text file named largeFile_1MB.txt (approximately 1MB in size).

2.Run the script:
python rsa-2048.py

3.The script will:
Generate a 2048-bit RSA key pair and print the generation time.
Encrypt the small and large files, producing smallFile_1KB_Enc.txt and largeFile_1MB_Enc.txt, and print encryption times.
Decrypt the encrypted files, producing smallFile_1KB_Dec.txt and largeFile_1MB_Dec.txt, and print decryption times.
Note: Times are printed in seconds but labeled as microseconds in the output—this is a labeling error. To get actual microseconds, multiply the printed value by 1,000,000.Script Overview
Key Generation: Uses RSA.generate(2048) to create the key pair.
Encryption: Reads files in 128-byte chunks, encrypts each with PKCS1_OAEP using the public key.
Decryption: Reads encrypted files in 256-byte chunks, decrypts each with PKCS1_OAEP using the private key.
Limitations
RSA is inefficient for large files; consider hybrid encryption (e.g., AES + RSA) for production use.
No error handling for file I/O or decryption failures.
Assumes input files exist; the script will fail otherwise.
